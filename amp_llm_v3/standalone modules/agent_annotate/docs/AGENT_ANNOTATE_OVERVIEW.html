Amphoraxe 5

Two-Pass Annotation Design

Every LLM-based agent uses Pass 1 (extract facts) → Pass 2 (apply decision tree)

Pass 1: Fact Extraction

LLM extracts structured facts from evidence:

  • Molecular class (peptide? protein? small molecule?)
  • Database hits (UniProt, DRAMP, ChEMBL)
  • Mechanism of action
  • Product description
  • Active ingredient role

Output is parsed into structured dict before Pass 2

Pass 2: Decision Tree

LLM applies explicit rules to extracted facts:

  • Step 1: Is the molecular class a peptide?
  • Step 2: Is the peptide the active drug?
  • Step 3: Database confirmation

Prevents 8B models from shortcutting on surface keywords

Post-Pass-2 check If Pass 1 says peptide but Pass 2 says False → trust extraction

Only citable facts. Rejects reasoning not grounded in evidence.

Adversarial

phi4-mini:3.8b

Challenges obvious answers. Tests if the annotation withstands scrutiny.

Model-Grouped Verification (v11 optimization) For each verifier model (3 total): Load model ONCE → verify ALL fields across ALL trials in batch → unload Before: 15 model switches per batch (field × verifier) After: 3 model switches per batch (one per verifier) Consensus: 2 of 3 verifiers agree → accepted Disagree → Reconciler (qwen2.5:14b) makes final call

Reasoning Patterns15%Generalized rules from corrections
Prompt Hints10%A/B tested prompt variations
Buffer10%Reserved

Epoch Decay

Old data from previous code versions is downweighted:

  • Current epoch: 100% weight
  • 1 epoch back: 75% (experiences), 80% (corrections)
  • 2 epochs back: 56% / 64%
  • Floor: 5% (never fully forgotten)

New code version = new epoch. Prevents stale knowledge from contaminating new behavior.

Amphoraxe 12

EDAM Self-Audit

Post-annotation consistency checking against the agent's own research evidence

4 Audit Types

  • Delivery Mode: checks OpenFDA route fields, protocol text for explicit route keywords
  • Peptide: checks UniProt AA length, database hits, semantic patterns. Requires 2+ signals for True→False corrections
  • Outcome: catches Positive without publications, missed Recruiting status, Unknown despite hasResults=true
  • Classification: catches missed AMP drugs classified as Other using database evidence

Key Principle

The agent never sees human annotations during learning.

It improves through evidence-grounded self-correction. The structured research data IS the ground truth.

Human concordance is measured independently as an evaluation metric, not a training signal.

Amphoraxe 13

Reasoning-First Improvement Strategy

Drug lists are frozen. All improvements through reasoning.

Layer 1: Research

Better evidence in → better answers out

  • Drug name resolution via LLM
  • Cached in EDAM after first hit
  • UniProt features (Chain/Peptide)
  • DRAMP/APD priority for sequences

Layer 2: Reasoning

Better LLM reasoning with evidence

  • Structured Pass 1→2 handoff
  • Per-field temperature tuning
  • Post-Pass-2 consistency check
  • Conservative bias (more False examples)

Layer 3: Validation

Cross-field structural rules

  • AMP → peptide=True
  • peptide=False → classification=Other
  • outcome → clears failure reason
  • Post-verification normalization

Design Principle: No hardcoded drug→classification lookup tables. The agent must reason from evidence. Drug lists are frozen at ~100 well-known drugs for performance only — future improvements come from better research, reasoning, and validation.
